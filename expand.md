1. IDs and classes matter because they are often the connection points between HTML, CSS, and JavaScript. A clear and consistent name makes it easier to style an element, select it in JavaScript, and understand its purpose later. Poor names can make code harder to maintain because a change in one file may unexpectedly break behavior in another file.

2. Data attributes are custom HTML attributes that start with `data-`, such as `data-user-id` or `data-status`. They are useful for storing small pieces of extra information directly on an element without inventing nonstandard attributes. In JavaScript, they can be accessed through the element's `dataset` property, such as `element.dataset.userId`. They should not be used as a replacement for semantic metadata like microdata, because data attributes are mainly for private page logic and do not give search engines or other tools the same structured meaning.

3. A DOM fragment is a lightweight container for DOM nodes that is not part of the live document tree until it is inserted. DOM fragments are powerful because you can build several elements inside the fragment first, then add them to the page all at once. This can make DOM updates cleaner and more efficient than repeatedly modifying the live page.

4. The point of a Virtual DOM is to keep a JavaScript representation of the UI, compare changes there, and update the real DOM only where needed. You gain a more declarative way to build interfaces and can avoid some manual DOM manipulation. You lose some direct control and add framework overhead, so for small pages vanilla DOM code can be simpler.

5. The `class` attribute is accessed as `className` in JavaScript because `class` is a reserved keyword in the language. To avoid that conflict, the DOM API exposes the HTML class attribute through the `className` property. Modern JavaScript also provides `classList`, which is often better for adding, removing, or toggling individual classes.

6. `addEventListener()` lets you attach one or more event handlers to the same element without overwriting other handlers. It is flexible and usually the better choice for larger code because different parts of the code can listen for the same event. An `onclick` property is simpler for small examples, but assigning a new value replaces the previous handler. Inline `onclick` in HTML can also mix structure and behavior, which usually makes code harder to maintain.
